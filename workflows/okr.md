# OKR Workflow

## Input
The quarter and what the team is focused on. Freeform. If unclear, ask: "What's the one thing that would make this quarter a success — in one sentence?"

## OKR rules
- One objective per team or product area (inspiring direction, not a task)
- 3–4 key results per objective (measurable, time-bound, outcome not output)
- Key results should be uncomfortable: 70% achievement = success, not failure
- For each KR: define a leading indicator — something you can track weekly, not just at quarter end
- If a KR starts with a verb like "ship" or "build", it's an output — rewrite it as an outcome

## Output

Save to `docs/okr-[YYYY-Qn].md` (e.g. `docs/okr-2026-Q2.md`):

## Check-in cadence
- **Weekly:** Review leading indicators only
- **Mid-quarter:** Assess if KR targets are still achievable — adjust if needed
- **End of quarter:** Score each KR (0.0–1.0), write a 3-sentence retro
